# Antigravity Project Instructions

## Project Name

**Fantasy Golf League**

---

# 1. Overview

Build a **Progressive Web App (PWA)** that allows users to run **season-long fantasy golf leagues** similar to fantasy football.

Users will:

* Create or join leagues
* Draft PGA golfers before the season
* Set weekly lineups for tournaments
* Earn fantasy points based on golfer performance

Each tournament acts as a **weekly matchup** where teams accumulate points.

The system will use **CSV files for golfer lists** and a **simple scraper to update round stats**.

# 4. Golfer Data System

Golfers will be imported from a CSV file.

File name:

```
master_golfer_list.csv
```

Structure:

```
golfer_id,golfer_name
1,Scottie Scheffler
2,Rory McIlroy
3,Jon Rahm
```

The app should import this CSV into the **golfers table**.

Database table:

```
golfers
id
golfer_name
```

File name example:

```

```

Structure:

# 6. Simplified Scraper

The scraper only needs to collect **basic round statistics**.

Stats to collect:

```
round_score
total_birdies
total_eagles
```

These stats will be stored in the table:

```
golfer_round_stats
```

Table structure:

```
id
tournament_id
golfer_id
round
score
birdies
eagles
updated_at
```

Example row:

```
tournament_id: 2
golfer_id: 1
round: 1
score: 67
birdies: 6
eagles: 1
```

---

# 7. Scraper Worker

Create a worker:

```
scrapeRoundStats.ts
```

Purpose:

* Fetch leaderboard data
* Extract stats for each golfer
* Update the database

Scraper flow:

```
fetch leaderboard
↓
loop through players
↓
extract score, birdies, eagles
↓
upsert golfer_round_stats
```

Pseudo-logic:

```
for each player in leaderboard:
  save
    tournament_id
    golfer_id
    round
    score
    birdies
    eagles
```

---

# 8. Scraper Schedule

Run the scraper **once after each round**.

Schedule:

```
Thursday evening
Friday evening
Saturday evening
Sunday evening
```

The scraper should update the stats using **upsert operations** to prevent duplicates.

---

# 9. Fantasy Scoring

Fantasy points will be calculated in the UI using the stored stats.

Example scoring system:

```
Birdie = +3 points
Eagle = +8 points
Stroke under par = +1 point
```

Example:

```
Score: 67 (-5)
Birdies: 6
Eagles: 1
```

Points:

```
6 birdies = 18
1 eagle = 8
5 under = 5

Total = 31 points
```

The UI should calculate team totals by summing points for **active golfers in the weekly lineup**.

---

# 10. Team Leaderboard Logic

For each team:

```
↓

↓

```

This leaderboard should update automatically when stats update.

# 15. Important Rules

Follow these rules:

* Use `golfer_id` as the primary identifier for golfers
* Do not rely on name matching
* Always use **upsert operations** when importing data
* The database should store **raw stats only**
* Fantasy scoring should happen in the frontend
